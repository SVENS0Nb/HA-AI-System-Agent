HA AI System Agent

Signal, OpenAI und Überwachung zentral in Home Assistant konfigurieren.

Status wird geladen …

    OpenAI

    Wird nie an die Oberfläche zurückgegeben.
    Automatisch verwendet lokal none bis high und benötigt keine zusätzliche API-Anfrage.
    Wird nur bei „Feste Stufe“ verwendet.

    Signal

    Signal-Bridge wird geprüft …Keine IP-Adresse oder separater Container erforderlich.

    In der Signal-App des gewünschten Kontos unter Einstellungen → Verknüpfte Geräte scannen.

    Sende von deinem persönlichen Signal-Konto exakt:

    Der einmalige Code ist fünf Minuten gültig. Die Absendernummer wird danach automatisch freigegeben.
    Beim Ändern der URL wird ein alter Proxy-Token sicherheitshalber verworfen.
    Nur für einen vorgeschalteten Bearer-Auth-Proxy.
    Im integrierten Modus automatisch erkannt.
    Standardmäßig ausgeschaltet. Nur eindeutig an die eigene Nummer gesendete Selbstnachrichten werden verarbeitet; normale Unterhaltungen bleiben auf die Absenderliste begrenzt.
    Gekoppelte Nummern dürfen mit dem Agenten chatten. Zusätzliche Nummern können weiterhin manuell ergänzt werden.

    Laufzeit und Speicher

    Gültige IANA-Zeitzone für Cron-Jobs und zeitabhängige Überwachungen.
    Erkennt verpasste Zustandswechsel nach Verbindungsabbrüchen.

    Lernen und Wissensbasis

    Zustandsereignisse werden lokal zu kompakten Baselines zusammengefasst. OpenAI wird erst bei einer belastbaren Auffälligkeit oder einer passenden Anfrage verwendet.
    Der Agent kann für einzelne Erinnerungen eine kürzere Lebensdauer wählen.
    Kontrolliertes Gedächtnis

    Nur exakte Aussagen aus deiner aktuellen Signal-Nachricht dürfen als Nutzererinnerung gespeichert werden. Logs, Configs und Events können keine Erinnerungsanweisungen einschleusen.

    Geräte- und Entity-Steuerung

    Standardmäßig ausgeschaltet. Der Agent darf nur eine fest definierte Auswahl physischer Entity-Aktionen vorschlagen; jede Ausführung benötigt anschließend einen separaten BESTÄTIGEN …-Befehl über Signal.
    Eine Entity-ID pro Zeile. Bei aktiver Steuerung ist mindestens ein Eintrag erforderlich; ein leeres Feld erlaubt keine Geräteaktion.
    Keine allgemeine Service-Freigabe

    Automationen, Skripte, Szenen, Buttons, Helfer, Alarmanlagen, Updates, System-/Add-on-Dienste und Dateiänderungen bleiben gesperrt. Sichere Geräte wie Schlösser oder Sirenen sollten zusätzlich über die Entity-Liste eingeschränkt werden.

    Sicherheit und Datenschutz

    Konfiguration und System bleiben read-only.

    Monitoränderungen und freigegebene Geräteaktionen werden erst nach einem separaten BESTÄTIGEN … im Signal-Chat aktiv.

    Nicht empfohlen. Auch bei Freigabe werden erkannte Kennwörter, Tokens und Schlüssel lokal geschwärzt, bevor Daten an OpenAI gehen.

    Diese Oberfläche prüft serverseitig, ob der angemeldete Home-Assistant-Benutzer Administrator ist.

    Speichern und testen

    Änderungen werden in /data/ui-settings.json gespeichert und ohne Add-on-Neustart geladen.
